Resumen Ejecutivo

Se desarrolló desde cero un panel de administración completo para el FAB LAB INACAP, reemplazando procesos manuales dispersos por un sistema centralizado, accesible desde cualquier dispositivo y con capacidad de instalación como aplicación nativa.

El sistema integra Google Sheets (registro de visitas), Supabase (base de datos en tiempo real), tecnología NFC (identificación de activos físicos) y un diseño mobile-first que permite operar el laboratorio desde el celular como si fuera una app nativa.

Tecnologías principales
  • Supabase — base de datos y autenticación
  • Google Apps Script — integración Sheets
  • Web NFC API — escaneo de tags físicos
  • Netlify — hospedaje y funciones serverless
  • PWA — instalable como app nativa

Calidad y Pruebas
Revisión exhaustiva pre-entrega

Antes de la entrega se realizó una revisión de código a máximo esfuerzo con 5 agentes de análisis independientes ejecutados en paralelo, cubriendo 10 ángulos distintos: seguridad XSS, validación de datos, condiciones de carrera, patrones peligrosos de JavaScript y eficiencia.

Resultado del QA
  • 15 bugs identificados y corregidos
  • 4 vulnerabilidades XSS eliminadas
  • 6 errores de lógica de negocio
  • 5 mejoras de robustez aplicadas
  • 0 bugs críticos en producción
Seguridad — XSS (4 fixes)
  • Toasts: mensajes de error escapados con escapeHTML()
  • Inventario: URLs de imagen en atributos src escapadas
  • Ayudantes: fecha_ingreso sin escapar en modal de perfil
  • escapeHTML incluye comilla simple para atributos onclick
Lógica de negocio (6 fixes)
  • Devolución: NaN pasaba validación qty y corrompía stock
  • Préstamo: campo vacío enviaba null zeroeando el stock
  • NFC modal: AbortController anterior no se abortaba al re-escanear
  • checkSession: error de BD causaba cierre de sesión sin aviso
  • doLogout: try/finally garantiza reload aunque signOut falle
  • saveNFCRegistration: retorno silencioso cuando activo no existe
Robustez del sistema (5 fixes)
  • checkDBStatus movido a post-login (evita falso error al ingresar)
  • db.ayudantes declarado en objeto inicial (no parcheado a posteriori)
  • parseInt con radix 10 en 6 funciones (previene error octal)
  • updateNFCStats con null-guards en elementos del DOM
  • @keyframes slideUp movido fuera del @media query
Buenas prácticas aplicadas
  • Todos los parseInt con radix explícito
  • Feedback visible al usuario en todos los flujos de error
  • Service Worker registra advertencias en consola
  • Validación de cantidad en préstamos con guard !qty
  • Estructura del objeto db declarada completamente al inicio

Experiencia Móvil y PWA
Diseño tipo aplicación nativa

El panel se comporta como una app nativa en dispositivos móviles, eliminando la fricción del navegador web.

📱
Barra de navegación inferior
Panel, Visitas, Inventario, NFC y Más — como WhatsApp o Instagram. El menú hamburguesa desaparece en móvil; todo accesible con el pulgar.
⬆️
Drawer "Más" tipo sheet
Se desliza desde abajo mostrando usuario, rol y acceso a todas las secciones secundarias. Animación nativa slide-up.
🗂️
Modales como bottom sheets
Todos los modales abren desde la parte inferior con bordes redondeados y handle bar — patrón estándar de iOS y Android.
🏠
Instalable en pantalla de inicio
Android: Chrome → "Agregar a pantalla de inicio". iOS: Safari → Compartir → "Agregar al inicio". Aparece con ícono y nombre propio.
📵
Funciona sin conexión
Service Worker cachea HTML y CSS. La interfaz carga incluso sin internet; los datos requieren conexión para sincronizarse con Supabase.
⚡
Optimizada para tacto
Botones de mínimo 48px, sin delay táctil, sin highlight azul en toque, inputs que no hacen zoom en iOS. Área segura en notch/punch-hole.
